# 博客文章写作指南

本目录存放所有博客文章。每篇文章是一个 **Markdown 文件**（`.md`），保存后网站会自动读取并展示。

**说明：** 本文件 `README.md` 是写作指南，**不会**作为文章发布。以 `_` 开头的文件（如 `_draft-xxx.md`）视为草稿，同样不会出现在网站上。

---

## 快速开始（3 步）

### 1. 新建文件

在 `posts/` 目录下创建文件，例如：

```
posts/my-new-article.md
```

**文件名规则：**

| 规则 | 说明 |
|------|------|
| 只用英文、数字、连字符 | 如 `tech-nextjs-tips.md`，不要用中文或空格 |
| 文件名 = 网址 slug | `tech-nextjs-tips.md` → 访问地址 `/blog/tech-nextjs-tips` |
| 后缀必须是 `.md` | 其他格式不会被识别 |
| 不要以 `_` 开头 | `_draft.md` 等草稿文件会被忽略，方便本地写作 |
| `README.md` 不算文章 | 本指南文件不会被收录 |

### 2. 填写头部信息（frontmatter）

文件**最开头**用 `---` 包裹 YAML 元数据，用来设置**标题、日期、分类、摘要**。

`title`、`date`、`category` 为**必填**；缺一项或格式不对时，`npm run build` 会直接失败并提示具体文件名和字段，避免上线后出现空白页。

```markdown
---
title: "文章标题（会显示在页面顶部和列表里）"
date: "2026-06-08"
category: "技术栈"
description: "一句话摘要，显示在博客卡片、搜索结果和 RSS 里。"
---

正文从这里开始写……
```

### 3. 写正文并发布

```bash
# 本地预览
npm run dev
# 浏览器打开 http://localhost:3000/blog 查看

# 确认无误后推送到 Git，Vercel 会自动部署
git add posts/my-new-article.md
git commit -m "新文章：xxx"
git push
```

## 头部字段说明

| 字段 | 必填 | 作用 | 出现在哪里 |
|------|------|------|------------|
| `title` | ✅ | 文章标题 | 文章页大标题、博客列表、首页「最新文章」、归档、搜索、RSS |
| `date` | ✅ | 发布日期 | 文章页、列表排序（新 → 旧）、归档按年月分组 |
| `category` | ✅ | 分类（即「标签」） | 文章页、列表筛选、标签页 `/blog/tags` |
| `description` | 建议填写 | 摘要 | 列表卡片、搜索预览、SEO / Open Graph、RSS |

构建时会校验：`title`、`date`、`category` 必须为非空字符串；`date` 还需能被解析为有效日期（如 `2026-06-08`）。`description` 可省略，省略时摘要为空。

### 标题（title）

- 写在 frontmatter 里，**不是**正文第一个 `# 标题`
- 建议用引号包裹，避免冒号等特殊字符导致 YAML 解析出错
- 正文里可以用 `##`、`###` 写小节标题，与页面大标题无关

```markdown
---
title: "Next.js 实战小技巧"
---

## 服务端组件与客户端组件   ← 这是正文小节，不是页面标题
```

### 日期（date）

- 格式：`"YYYY-MM-DD"`，建议加引号
- 必须是浏览器能解析的有效日期，否则构建失败
- 列表按日期**倒序**排列，日期越新越靠前
- 归档页按 `date` 的年月分组（如 `2026-06`）

### 分类（category）

本博客的「分类」= 标签页里的「标签」，**一篇文只有一个分类**。

当前已有分类示例：

- `技术栈` — 技术实践、踩坑、工具用法等
- `生活感悟` — 非技术类随笔

**新增分类：** 直接在 `category` 里写新名字即可，无需改代码。保存后它会自动出现在：

- 博客列表顶部的分类筛选按钮
- `/blog/tags` 标签页

**注意：** 同一分类请保持名称完全一致（包括空格、大小写），否则会被当成两个分类。例如 `技术栈` 和 `技术 栈` 是不同分类。

### 摘要（description）

- 一两句话概括文章，会显示在卡片和 RSS 里
- 搜索也会匹配摘要内容
- 建议 30～80 字，说清楚「这篇文章讲什么」
- 非必填，但强烈建议填写，否则列表卡片和 RSS 摘要会为空

## 草稿写法

还没写完、不想发布时，文件名以 `_` 开头即可，例如：

```
posts/_my-draft.md
```

草稿不会被网站读取，frontmatter 也可以先留空。定稿后去掉 `_` 前缀并补全必填字段，再提交发布。

## 正文 Markdown 支持

博客使用 `react-markdown`，支持常见 Markdown 和 GFM 扩展：

| 语法 | 示例 |
|------|------|
| 标题 | `## 二级标题`、`### 三级标题` |
| 粗体 / 斜体 | `**粗体**`、`*斜体*` |
| 链接 | `[文字](https://example.com)` |
| 图片 | `![说明](图片地址)` |
| 引用 | `> 引用内容` |
| 列表 | `- 无序` 或 `1. 有序` |
| 表格 | GFM 表格语法 |
| 代码 | 行内 `` `code` `` 或围栏代码块（见下） |
| 分隔线 | `---`（三个短横线，单独一行） |

### 代码块

支持语法高亮，代码块右上角有「复制」按钮：

````markdown
```ts
export function hello() {
  return "world";
}
```
````

常用语言标识：`ts`、`tsx`、`js`、`bash`、`json`、`css`、`html` 等。

## 各字段在网站上的展示位置

```
posts/xxx.md
    │
    ├── 首页 /                    → 最新 3 篇（按 date 排序）
    ├── /blog                     → 全部文章列表 + 分类筛选
    ├── /blog?category=技术栈     → 按分类筛选
    ├── /blog/tags                → 所有分类（标签）及文章数
    ├── /blog/archive             → 按年月归档
    ├── /blog/search?q=关键词     → 搜索 title / description / 正文
    ├── /blog/xxx                 → 文章详情页
    └── /feed.xml                 → RSS 订阅（title + description + 链接）
```

## 修改与删除

### 修改已有文章

直接编辑对应的 `.md` 文件即可。改 `title`、`category` 等 frontmatter 会同步更新所有展示位置。

### 删除文章

删除 `posts/` 下对应的 `.md` 文件，重新部署后该文章会从网站消失。

### 改文件名（改 URL）

例如把 `old-name.md` 改成 `new-name.md`：

- 旧链接 `/blog/old-name` 会 404
- 新链接变为 `/blog/new-name`
- 若文章已对外分享，建议在旧文末尾留一句「已迁移至 xxx」或做跳转（需改代码）

## 发布检查清单

发布前过一遍：

- [ ] 文件名是英文 slug，无空格和中文，且不以 `_` 开头
- [ ] frontmatter 中 `title`、`date`、`category` 已填写且非空
- [ ] `date` 格式正确（`YYYY-MM-DD`），且为有效日期
- [ ] `description` 已写好（建议）
- [ ] `category` 名称与预期一致（无多余空格）
- [ ] 本地 `npm run dev` 预览正常
- [ ] 本地 `npm run build` 通过（会校验所有文章的 frontmatter）
- [ ] 文章页标题、分类、日期显示正确
- [ ] 博客列表里能搜到 / 筛到
- [ ] `git push` 后等 Vercel 部署完成，线上再确认一次

## 常见问题

**Q：文章写了但列表里看不到？**

- 确认文件在 `posts/` 目录，后缀是 `.md`
- 确认文件名不是 `README.md`，且不以 `_` 开头
- 确认 frontmatter 以 `---` 开头和结尾，且 `title`、`date`、`category` 均已填写
- 运行 `npm run build`，查看是否报 `[posts] xxx.md: frontmatter …` 类错误
- 重启 dev 服务器：`Ctrl+C` 后重新 `npm run dev`

**Q：`npm run build` 报错 `[posts] … frontmatter`？**

- 按报错中的文件名和字段名检查 frontmatter
- 常见原因：漏写 `title` / `date` / `category`、字段值为空、或 `date` 写成了无效日期
- 修好对应 `.md` 文件后重新构建即可

**Q：标题显示不对？**

- 页面大标题来自 frontmatter 的 `title`，不是正文 `# 标题`
- 检查 `title` 是否在 `---` 之间，值是否加了引号

**Q：分类筛选点不到 / 标签页没有？**

- 检查 `category` 是否填写且非空
- 分类名前后不要有多余空格

**Q：首页「最新文章」没更新？**

- 首页只显示**日期最新**的 3 篇，检查 `date` 是否比其他文章更新

**Q：RSS 链接不对？**

- 在 Vercel 配置环境变量 `NEXT_PUBLIC_SITE_URL` 为你的线上完整地址（含 `https://`），详见项目根目录 `DEPLOY.md`
